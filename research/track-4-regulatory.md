# Track 4 — 규제 경계 (공개 가이드라인 기준)

> 본 문서는 **법률 자문이 아니며**, 식품의약품안전처(MFDS), 보건복지부(MOHW),
> 개인정보보호위원회(PIPC), 국가법령정보센터(law.go.kr) 등 **공개 정부 자료**에서
> 확인 가능한 범위의 사실만 정리한 것입니다. 해커톤 예선 제안서(5페이지)에
> "규제 경계 인지"를 증빙하기 위한 것이며, 실제 출시 전에는 의료기기·약사·
> 개인정보 분야 전문가 자문이 반드시 필요합니다.
>
> 제품 설계 전제: 필케어(PillCare)는 (1) 환자에게 **복약 리마인드·의약품 정보
> 제공**, (2) 상호작용·중복·주의사항이 감지되면 **"의료진/약사에게 확인하세요"**
> 라는 문서화된 초안을 만들어 환자가 의사에게 전달할 수 있도록 **브리지 메시지
> 초안**을 작성, (3) 본인/피부양자 프로필 관리 — **진단·용량 조절·복용 중단 판단
> 기능은 제공하지 않음**.

## 2. "필케어는 의료기기에 해당하는가?" 판단 체크리스트
(식약처 「의료기기와 개인용 건강관리(웰니스) 제품 판단기준」 공개 자료 기반)

- [x] **진단 목적인가?** → **No** — 질병 진단·검사 결과 해석 기능 없음
- [x] **치료 목적인가?** → **No** — 약물 투여 제어, 증상 개선 직접 개입 없음
- [x] **증상의 경감·보정·예방 목적인가?** → **No (조건부)** — "복약 시간 리마인드"는 일상적 건강관리 범주. 단, "이 약을 먹지 마세요"/"용량을 줄이세요" 같은 처방 변경 제안은 **절대 금지**
- [x] **약물 용량을 산출·제안하는가?** → **No** — LLM 출력 allowlist로 용량 계산/제안 문장 차단
- [x] **의료진의 임상 의사결정을 직접 지원하는가?** (CDSS) → **No** — 필케어의 사용자는 환자 본인/보호자이며, 의사·약사용 의사결정 지원 UI 없음. "의사에게 전달할 메시지 초안"은 의사결정을 대신하는 것이 아니라 환자가 의사에게 **질문**을 구성하도록 돕는 문서화 보조
- [x] **의약품·의료기기 오작동/부작용 감시 기능?** → **No** — 이상반응이 보고되면 "의료진 확인" 플로우로 귀결, 필케어가 스스로 "이것은 부작용입니다"라고 단정하지 않음
- [x] **위해도**(식약처 판단기준의 핵심 축): 필케어 출력이 환자에게 잘못 받아들여져도, **최종 결정이 환자·의료진에게 남는 구조**이면 위해도 낮음. 단, "상호작용 경고"의 톤이 단정적이면 위해도↑ → UX 가드레일이 규제 경계의 핵심

- **잠정 결론**: 식약처 판단기준의 "개인용 건강관리(웰니스) 제품" 정의("건강 상태 또는 건강한 활동의 유지·향상 목적, 만성질환 자가관리" 범주)에 부합 — **비의료기기(웰니스)로 해석 가능**. 단, 다음 세 가지를 지키지 못하면 의료기기법 적용 리스크가 급격히 상승한다:
  1. 용량·복약 중단·대체약 제안 금지
  2. "진단/처방/치료" 단어 및 유사 프레이밍 금지
  3. 모든 경고의 결론은 "의료진 확인 요청"으로 종결
- **권고**: 론칭 전 식약처 **의료기기정책과 비공식 사전상담** 또는 **한국의료기기안전정보원** 문의로 "민원인 질의회신" 확보 (공개 절차).

## 4. 남은 불확실성 / 전문가 자문 필요 사항

1. **"상호작용 경고"의 SaMD 해당 여부 임계선** — 식약처 공개 지침은 위해도 기반이며, "정보 제공"과 "의료기기 기능"의 경계가 제품 UX에 따라 달라진다. 릴리스 전 식약처 민원인 질의회신 또는 한국의료기기안전정보원 사전상담 권장.
2. **생성형 AI 출력의 책임 귀속** — 2025 생성형 AI 의료기기 가이드라인이 공개되었으나, 비의료기기 소비자 앱에서 LLM이 환자에게 제공하는 정보의 오류·환각에 대한 제조자 책임은 민사상 문제로 남음. 제조물책임법/약관·면책고지 설계 전문가 자문 필요.
3. **DUR 데이터 사용 가능성** — 심평원 DUR은 의사·약사의 처방/조제 시점에 작동하는 제도이며, 환자용 앱이 동일 DUR 데이터베이스를 직접 사용할 수 있는지는 공개 자료로 확인 불가. 심평원 문의 필요.
4. **국외 이전** — LLM이 해외 서버(예: Anthropic/OpenAI US 리전)로 민감정보가 흘러갈 경우, 개인정보 보호법 제28조의8(국외 이전) 기준 별도 동의 및 고지가 필요. 국내 리전/온디바이스 옵션 검토 필요.
5. **피부양자(노부모) 프로필** — 법정대리인 동의 구조는 만 14세 미만에 대해서만 명문화되어 있고, 성인 피부양자(치매 부모 등)는 위임·대리 동의 구조가 별도 법령(민법상 대리, 의료법상 대리 동의)에 걸쳐 있어 단순화된 UX 설계가 어렵다. 전문가 자문 필요.
6. **"브리지 메시지" 문서의 법적 성격** — 환자가 의사에게 전달하는 "질문 초안"이 진료 현장에서 의사결정에 영향을 미쳤을 때, 기록의 법적 성격(의료기록? 환자 진술?)에 대한 판례는 공개 자료로 확인 불가.

## 6. 설계 반영 체크리스트 (제안서 "실용성·책임감" 점수 방어용)

- [ ] LLM 시스템 프롬프트에 **금지 어휘 리스트**(진단·처방·복용 중단·용량 조절·대체약)
- [ ] 모든 경고 응답의 **결론 문장 템플릿** = "의료진/약사에게 확인하세요"
- [ ] 민감정보(건강) 수집에 대한 **별도 동의** UI + 개인정보 처리방침의 민감정보 섹션
- [ ] 만 14세 미만 자녀 프로필은 **보호자 법정대리인 동의** 후 생성
- [ ] 성인 피부양자(부모) 프로필은 본인 동의 또는 위임 근거 기록
- [ ] 국외 이전이 발생하면 **별도 고지·동의**; 가급적 국내 리전/온디바이스 옵션
- [ ] 제품 문구/마케팅에 "진단·치료·처방" 단어 금지
- [ ] 이상반응·위해 발생 시 **식약처/의료기기안전정보원 신고 경로** 안내
- [ ] 출시 전 식약처 민원인 질의회신(비의료기기 확인) 확보 계획
